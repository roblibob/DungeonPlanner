# Game Master Guide

This guide is for GMs building and presenting encounters in DungeonPlanner.

## Core prep workflow

1. Paint room layout with the **Room** tool.
2. Add doors/openings with the **Opening** tool.
3. Place props and lights with the **Prop** tool.
4. Organize with **rooms**, **layers**, and **floors**.
5. Save your dungeon JSON for session reuse.

## Fast map-building tips

- Block out shape first; detail second.
- Use layers for optional reveals and variants.
- Keep naming consistent (floors, rooms, key props).
- Use camera presets intentionally:
  - **Perspective** for cinematic prep
  - **Isometric** for structure checks
  - **Top-down** for tactical readability

## Multi-floor encounters

- Use floor levels (`0`, `+1`, `-1`) to structure vertical maps.
- Use staircase assets to connect navigation intuitively.
- Switch floors during play with clear callouts.

## Presentation quality checklist

- Ensure key paths are readable from top-down.
- Verify openings are placed where traversal is intended.
- Keep clutter moderate so tokens and routes stay visible.
- Check lighting balance before play (avoid over-dark or over-bright scenes).

## Save/load

- Save often as `.dungeon.json`.
- Keep milestone versions before major edits.
- If collaborating, share the file and note intended camera mode/layers.
