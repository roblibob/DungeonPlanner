# Contributing to DungeonPlanner

Thanks for considering a contribution to **DungeonPlanner**.
This project is built in the open, and we welcome bug reports, fixes, and improvements.

---

## Code of Conduct

By participating, you agree to follow our [Code of Conduct](./CODE_OF_CONDUCT.md).

---

## How to Contribute

You can help by:

* Reporting bugs via [issues](https://github.com/finger-gun/DungeonPlanner/issues)
* Proposing features and UX improvements
* Improving documentation
* Submitting code changes with tests

---

## Development Setup

1. Fork and clone:

   ```bash
   git clone git@github.com:your-username/DungeonPlanner.git
   cd DungeonPlanner
   ```

2. Install dependencies:

   ```bash
   pnpm install
   ```

3. Run the app:

   ```bash
   pnpm run dev
   ```

4. Run checks before opening a PR:

   ```bash
   pnpm run verify
   ```

---

## Pull Requests

* Keep PRs focused and scoped.
* Link related issues when relevant.
* Add or update tests for behavior changes.
* Ensure local checks pass before submitting.

---

## Commit & Branch Guidelines

* Branch from `main`:

  ```bash
  git checkout -b feature/my-change
  ```

* Use clear commit messages that explain intent.
* Rebase or merge `main` as needed to resolve conflicts.

---

## Questions

If you are unsure about an approach, open an issue or a draft PR early.
